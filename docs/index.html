AI Workflow Comparison

Innerloop vs Direct vs ShipIt — SVP Briefing — 2026-05-17

Executive Summary

Four real-world scenarios tested across Innerloop, Direct (Claude Code bare), and ShipIt. Each scenario used identical prompts, same codebase, same hardware. A 5-model SFDC Council independently scored outputs.

Top 3 Findings

Speed and quality favor Direct — Direct is fastest on 3 of 4 scenarios and achieved the highest average council score (20.3/25). Ideal for GUS work items and time-boxed tasks.

Innerloop produces the most tests — Across all scenarios, Innerloop wrote significantly more tests. In S4, Innerloop wrote 57 tests vs Direct's 31 and ShipIt's 23. Best choice when test coverage is a hard requirement.

ShipIt anomaly requires attention — In S2, ShipIt's first run produced 0 code changes (cause unknown). A re-run succeeded. Always validate ShipIt output before merging. ShipIt is not broken — this was an isolated anomaly.

Key Cost Insights

Cost differences are negligible. The spread between cheapest (Direct: $16.48) and most expensive (Innerloop: $28.73) is ~$12.25 — a 74% difference. Choose workflow based on quality and test coverage needs, not cost.

S3 is the most expensive scenario — The GUS work item required more token-intensive reasoning and test generation across all 3 workflows ($3.64–$4.60 each).

Innerloop is costlier on open-ended tasks — Its structured review phases consume more tokens on exploratory work (S1: $5.08 vs Direct $4.13). On constrained bug-fix tasks (S2), all three are similar ($4-$8).

Recommendations

Use Innerloop when...

  • Production code going to main
  • Test coverage is a hard requirement
  • You want a built-in council review gate
  • RAG or advanced integration needed
  • Sprint features with Definition of Done

Use Direct when...

  • Speed is the priority (<15m window)
  • GUS work item with clear acceptance criteria
  • Highest quality output per dollar
  • Bug fix with known root cause
  • Demo or prototype code

Use ShipIt when...

  • Fully unattended pipeline required
  • Batch processing many small tasks
  • No human available in the loop
  • Always validate output before merging — silent anomalies are possible
